## Highlights

- Code on any device with a consistent development environment
- Use cloud servers to speed up tests, compilations, downloads, and more
- Preserve battery life when you're on the go; all intensive tasks run on your
  server

## Requirements

See [requirements](https://github.com/innovatex-official/IDE/docs/innovatex-ide/latest/requirements) for minimum specs, as well as instructions
on how to set up a Google VM on which you can install innovatex-ide.

**TL;DR:** Linux machine with WebSockets enabled, 1 GB RAM, and 2 vCPUs

## Getting started

There are five ways to get started:

1. Using the [install
   script](https://github.com/innovatex-official/IDE/blob/main/install.sh), which
   automates most of the process. The script uses the system package manager if
   possible.
2. Manually [installing
   innovatex-ide](https://github.com/innovatex-official/IDE/docs/innovatex-ide/latest/install)
3. Deploy innovatex-ide to your team with [innovatex/innovatex](https://cdr.co/innovatex-github)
4. Using our one-click buttons and guides to [deploy innovatex-ide to a cloud
   provider](https://github.com/innovatex/deploy-innovatex-ide) ⚡
5. Using the [innovatex-ide feature for
   devcontainers](https://github.com/innovatex/devcontainer-features/blob/main/src/innovatex-ide/README.md),
   if you already use devcontainers in your project.

If you use the install script, you can preview what occurs during the install
process:

```bash
curl -fsSL https://github.com/innovatex-official/IDE/install.sh | sh -s -- --dry-run
```

To install, run:

```bash
curl -fsSL https://github.com/innovatex-official/IDE/install.sh | sh
```

When done, the install script prints out instructions for running and starting
innovatex-ide.

> **Note**
> To manage innovatex-ide for a team on your infrastructure, see: [innovatex/innovatex](https://cdr.co/innovatex-github)

We also have an in-depth [setup and
configuration](https://github.com/innovatex-official/IDE/docs/innovatex-ide/latest/guide) guide.
